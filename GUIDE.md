# Virtual Try-On Guide

## What This Repo Is

This project is notebook-first. There is no `npm run`, `make`, or single training task file in the repo right now. The actual workflow is split across the notebooks, and the final interactive demo can be launched with the root `app.py`.

Use one of these paths:

1. Run the notebooks in order if you want the full step-by-step workflow.
2. Run `python app.py` if you want the Gradio UI directly.

## Recommended Environment

- Python `3.10` or `3.11`
- Windows PowerShell is fine
- NVIDIA GPU strongly recommended
- `12 GB` VRAM or more is much more comfortable for SDXL
- First run downloads several GB of model weights

CPU-only runs are possible for smoke testing, but they will be very slow.

## Setup

Open PowerShell in the repo root and run:

```powershell
cd C:\Users\ganes\Desktop\AI
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

Install PyTorch first. Pick the wheel that matches your machine:

```powershell
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

Then install the project dependencies:

```powershell
pip install -r requirements.txt
```

If you do not have an NVIDIA GPU, swap these packages after install:

- Replace `onnxruntime-gpu` with `onnxruntime`
- Replace `rembg[gpu]` with `rembg`

## Common Errors And Fixes

### `ModuleNotFoundError`

Your virtual environment is probably not active, or the dependency install did not finish.

Run:

```powershell
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### `No module named ...`

Same fix as above. Also make sure you installed PyTorch before `requirements.txt`.

### `xformers` error

You do not need `xformers` anymore for this repo. The notebooks and `app.py` now skip that optimization automatically if it is unavailable.

### `CUDA out of memory`

Try these first:

- Close other GPU-heavy apps
- Lower the number of denoising steps
- Disable upscaling
- Run one notebook at a time
- Restart the kernel before rerunning a notebook

### Missing files like `my_pose.png`, `my_identity.npz`, or `my_garment_segmented.png`

Those files are generated by earlier notebooks. Run the workflow in order.

### `git` says `detected dubious ownership`

That is a local Git safety check in the sandboxed environment. If you need Git commands locally, run:

```powershell
git config --global --add safe.directory C:/Users/ganes/Desktop/AI
```
